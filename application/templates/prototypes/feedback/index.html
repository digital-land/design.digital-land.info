User needs

  • As a GIS officer, I need to understand what DL means by a unique reference and why it’s important, so that I can find a suitable solution that works for me and my policy colleagues.
  • As a GIS/policy officer, I need to know the deadline for resolving an issue, to that I know how quickly I need to act.
  • As a GIS officer, I need to see the sources DL has, so that I can check them.
  • As a GIS/policy officer, I need to be notified of actions, so that I know when there are changes that need my attention.
  • As a GIS/policy officer, I need to resolve actions so that I don’t have to repetitively check (something that isn’t wrong).
  • As a planning policy officer, I need something unique to identify an error, so that I can find (the entity) within my own datasets.
  • As a GIS/policy officer, I need to visualise my data on a map, so that I can see the data and errors more quickly.
  • As a GIS/policy officer, I need to see which points on the map have errors or missing attributes, so that I can investigate them.

Things the prototype should include

  • Actions and recommendations per dataset
  • Map and text view or errors
  • Unique reference for entity that the user can use to search their own data
  • Dynamic errors/actions
  • Dynamic naming rather than entity

Iterations from v1 to v2

  • Remove summary section from all pages
  • Include timeframe for action to be taken
  • Separate actions that are needed vs recommendations within the data collection report
  • Compile / include a list of errors and make them more relevant to the ways users are providing data and the kinds of datasets
  • offer a way to ignore/remove something that isn't actually an error/problem (maybe an email to JIRA for now?)
  • Use whatever the 'unique ID' for an entity is, rather than address
  • Change map red dots to be all entities that need action rather than only entities outside of the LPA boundary
  • Potentially? combine the map view and text view of the data subset page
  • Get rid of the word entity and replace with dynamic word for each dataset, eg. TPOs, conservations areas, listed buildings
