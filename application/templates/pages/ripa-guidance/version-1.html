How to publish your data

The Digital Land platform is able to consume data in a variety of formats and structures providing that the data is on a URL that is publicly accessible.

Once we’ve configured the initial collection of data and tested that everything works as expected we’ll routinely attempt to collect any new or amended data that you have published.

Data specifications

For each dataset there are certain required fields as well as some additional information that will make the dataset more complete.

Providing that the fields are contained within the dataset you won’t need to rename the headers to match the specification, the Digital Land platform is able to manage that translation.

Article 4 Directions

Required fields

  • Reference – a reference or ID that is unique to each article 4 direction
  • Name – the name of the article 4 direction
  • Geometry – polygon coordinates for each article 4 direction site boundary
  • Start date – the date that the article 4 direction was officially designated

Additional fields

  • Point – if your GIS application provides additional point data then we can add this to the data we collect alongside the geometry
  • Document URL – the url to an authoritative document or web page that details the article 4 direction
  • End date – the data on which an article 4 direction ceased to be designated

Conservation areas

Required fields

  • Reference – a reference or ID that is unique to each conservation area
  • Name – the name of the conservation area
  • Geometry – polygon coordinates for each conservation area site boundary
  • Start date – the date that the conservation area was officially designated

Additional fields

  • Legislation – the act of legislation that was used to define the conservation area
  • Notes – any notes related to a conservation area
  • Point – if your GIS application provides additional point data then we can add this to the data we collect alongside the geometry
  • End date – the data on which an article 4 direction ceased to be designated
  • Document URL – the url to an authoritative document or web page that details the conservation area

Listed buildings and locally listed buildings

Required fields

  • Reference – a reference or ID that is unique to each listed building
  • Name – the name of the listed building
  • Geometry – polygon coordinates for each listed building boundary
  • Grade – the grade of the listed building
  • Start date – the date that the listed building was officially designated

Additional fields

  • Document URL – the url to an authoritative document or web page that details the listed building
  • Point – if your GIS application provides additional point data then we can add this to the data we collect alongside the geometry
  • End date – the data on which the listed building ceased to be designated

Listed buildings and locally listed buildings

Required fields

  • Reference – a reference or ID that is unique to each tree preservation order, tree or tree zone/area
  • Name – the name of the tree preservation order, tree or tree zone/area
  • Document URL – the url to an official document or web page that details the tree preservation order
  • Species – the species of the tree, or the species of tree within a tree zone/area
  • Tree preservation order type – whether the data relates to an individual tree or a tree zone/area
  • Geometry – polygon coordinates for each tree zone/area
  • Point – the point coordinates of an individual tree
