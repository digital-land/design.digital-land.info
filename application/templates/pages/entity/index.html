Entity page

{% set overviewHTML %}

User needs

  • As somebody looking for data I need to be able to trust the data planning.data.gov.uk provides is accurate, up-to-date and authoritative.
  • As a data provider I need to understand how the data we share is processed and used by the platform to trust that the data is still accurate and correct.

Hypothesis

For users to to trust the data planning.data.gov.uk provides we have the following hypothesis:

  • If we show users where this data came from (Which organisation(s) provided it) that will build trust in the data
  • If we show users where individual field values came from it will help them to understand the authoritative rules MHCLG applies and build trust in the data
  • If we show users how the data has changed over time, they will understand how the data is kept up-to-date and accurate which will increase trust in the data

Design questions

Is there a need for an interactive map if the 'entity' is a geography? Is a static render of the boundary/polygon enough?
